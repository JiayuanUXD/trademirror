## 一、产品愿景与设计哲学

### 1.1 一句话定位

> **TradeMirror 不帮你预测市场，它帮你看清自己。**

### 1.2 核心痛点（来自真实用户画像）

调研对象画像：6年A股投资者，本金100万，最高浮盈20%，最大回撤23%，目前浮亏18%。

| 痛点 | 现状 | 产品对策 |
|---|---|---|
| 情绪化交易 | 追高、急躁、被盘中波动绑架 | 决策卡强制流程 + 情绪三维评分 |
| 越跌越补 | 重仓股被套后加仓，扩大亏损 | 持仓集中度可视化预警 + 加仓规则锁 |
| 盈利不止盈 | 牛股变熊股，浮盈缩水成浮亏 | 移动止盈线条件单 + 浮盈缩水预警 |
| 重复犯错 | 6年下来同样的错犯了无数次 | "错误类型库"+ "新旧错误"自动归类 |
| 缺乏自我认知 | 凭感觉认为自己擅长某类操作 | 数据真相仪表盘（胜率、依据分布） |
| 目标不切实际 | 想短期回本→重仓→爆亏 | 合理目标设定向导 + 复利计算器 |

### 1.3 设计哲学（与TraderWaves的差异化）

| 维度 | TraderWaves | TradeMirror |
|---|---|---|
| 重心 | 交易数据分析 | 交易**行为**分析 |
| 用户对象 | 全球外汇/加密/股票交易者 | A股个人投资者 |
| 核心信仰 | 数据驱动改进策略 | **看见自己 → 改变行为 → 自然盈利** |
| 关键能力 | Analytics（盈亏分析、胜率） | **情绪追踪、纪律评估、错误归因** |
| 数据接入 | 全球券商API | A股券商导入 + 手动 + 通达信导出 |
| 独创点 | Backtester回测 | **核心问题对照表 + 错误库 + 移动止盈条件单** |

**核心原则**：**让用户痛**。看到自己FOMO评分7分时下的单大概率亏、连续违反纪律的周收益必然差、"凭感觉"决策的胜率不到30%——**只有数据揭穿幻觉，行为才会改变**。

## 二、目标用户与使用场景

### 2.1 用户画像（Persona）

#### 主要用户：阿强（重点服务对象）
- 38岁，男，IT工程师，业余炒A股
- 投资6年，本金80-150万
- 自评"会看基本面、能选对方向"
- 真实数据：6年累计负收益，因情绪化交易反复亏损
- 已尝试：纸质日记（坚持2周放弃）、Excel（公式太复杂）、Notion（搭建太繁琐）
- 核心诉求："我知道我有问题，但不知道问题具体是什么"

#### 次要用户：小敏
- 28岁，女，金融从业者
- 投资2年，本金20-40万
- 主要做ETF和大盘股
- 诉求：建立稳健的复盘习惯，避免成为下一个"阿强"

### 2.2 核心使用场景

**场景1：盘中下单前（高频）**
> 阿强看到某AI股冲高，手指准备点买入。打开App → 30秒决策卡 → 系统提示"FOMO评分7分+决策依据'看到别人在买'+不符合体系" → 弹出当头一棒：「⚠️ 你过去类似情况下12次操作有10次亏损，确定继续吗？」

**场景2：周日复盘（中频）**
> 周日晚9点，阿强收到推送"该做周复盘了"。15分钟完成本周数据回顾 + 自动生成的"本周高危交易"报告 + 三个关键问题。

**场景3：月末画像（低频但重要）**
> 月末App推送"本月画像生成完毕"。阿强看到一份自己的"行为肖像"：理性决策占比42%、最常犯的错误是"盈利不止盈"（已重复11次）、过去6个月该错误的代价累计是8.2万元。

**场景4：浮盈预警（被动触发）**
> 阿强持仓的烽火通信浮盈180%，App推送：「烽火通信浮盈达历史最高，过去用户达到180%浮盈未止盈的，3个月内65%回撤到100%以下」。

### 3.2 模块①：📝 交易决策卡（核心中的核心）

**功能定位**：所有交易行为的入口和出口。没有决策卡，就没有交易。

#### 3.2.1 创建决策卡（下单前必填）

**界面流程**（手机端为主，3屏完成）：

**屏1：基础信息（15秒）**
- 股票（从持仓库选 / 搜索输入）
- 方向（买入/加仓/卖出/减仓/清仓）
- 价格、数量（自动算金额）
- 一句话理由（限30字，超过强制截断）

**屏2：决策依据 + 情绪评分（30秒）**
- 决策依据（多选，6理性 + 4非理性）
  - 🟢 基本面变化 / 技术面信号 / 板块轮动 / 大盘环境 / 执行计划 / 消息面催化
  - 🔴 凭感觉 / 看到别人买卖 / 情绪驱动 / 想试试看
- **三维情绪评分**（滑块，1-10）
  - 平静度：现在内心平静吗？
  - 信心度：对这笔交易有多确定？
  - FOMO：怕错过的程度？

**屏3：风控与符合度（15秒）**
- 预设止损价（强制填）
- 最大可接受亏损金额（自动算 = (现价-止损价)×数量）
- 是否符合自己的交易体系？（符合/部分符合/不符合）

#### 3.2.2 智能拦截机制（重点）

提交决策卡时，根据填写内容触发**软拦截弹窗**（不是完全禁止，但要强制确认）：

| 触发条件 | 拦截提示 | 用户操作 |
|---|---|---|
| FOMO ≥ 7 | "你的FOMO评分较高。历史数据：FOMO≥7时你的胜率为28%（共14次操作）。要继续吗？" | 取消 / 我知道了，继续 |
| 平静度 ≤ 4 | "你目前心态不稳。建议把这笔交易加入观察清单，明天9:30前再决定。" | 加入观察 / 我知道了，继续 |
| "不符合"体系 | "这笔交易超出你的体系。本月不符合体系的操作已有3笔，2笔亏损。" | 取消 / 我知道了，继续 |
| 单次仓位>25% | "本笔操作将使该股仓位达到37%。你的纪律是单只≤25%。" | 减少数量 / 修改 / 强行继续 |
| 与最近一笔操作时间间隔<2小时 | "你2小时前刚做过一笔交易。频繁操作是你过去6年最常见的失误模式。" | 取消 / 继续 |

**关键设计**：弹窗中**显示用户自己的历史数据**——不是泛泛的警告，而是"**你过去**类似情况下**真实**发生了什么"。

#### 3.2.3 交易完成后（自动+手动跟踪）

- 成交价（券商导入或手填）
- 一周后：自动提醒回填当前价
- 一月后：自动提醒回填当前价，并写"事后复盘"
- 一月浮盈%自动计算

#### 3.2.4 决策卡字段总表

```
基础字段：编号、日期时间、股票、代码、方向、价格、数量、金额
决策字段：理由、决策依据（多选）、是否符合体系
情绪字段：平静度、信心度、FOMO（各1-10）
风控字段：预设止损价、最大可接受亏损
跟踪字段：实际成交价、一周后价、一月后价、一月浮盈%
反思字段：危险信号（系统自动）、事后复盘（用户写）
```

### 3.3 模块②：🎯 持仓与逻辑库

**功能定位**：每只持仓股票的"成长档案"。把"研究了再买"变成可验证的习惯。

#### 3.3.1 持仓总览页

卡片式列表，每只股票一张卡片，显示：
- 股票名称 + 代码
- 当前价、成本价、浮盈%
- 当前仓位% + 仓位条形图
- 板块标签
- 状态：🟢持有 / 🟡观察 / ⚫已清仓
- **健康分**（0-100，根据"逻辑完整度+撤退条件设置+操作纪律"综合计算）

#### 3.3.2 单股详情页（核心）

每只股票的详情页包含 5 个 Tab：

**Tab 1：📌 持有逻辑**
- 基本面理由（至少3条）
- 行业地位/护城河
- 关键财务数据（手填或抓取）
- **逻辑评分**（满分10分，每条理由是否有数据支撑、是否可验证）

**Tab 2：✅ 持有前提**（checkbox 清单）
- 半年报/年报业绩兑现
- 行业景气度不变
- 政策环境支持
- 自定义条件

**Tab 3：🚨 撤退条件**（任一触发即推送预警）
- 跌破止损价 ___ 元
- 单季度业绩增速 < ___%
- 跌破关键技术位 ___ 元
- 板块系统性回调（条件：行业指数跌破___）
- 出现重大利空（事件触发，需手动确认）

**Tab 4：📊 操作记录**
- 时间轴展示所有该股的决策卡
- 每笔操作的盈亏贡献
- "你在这只票上做对的事 vs 做错的事"自动总结

**Tab 5：🔍 新闻追踪**
- 手动粘贴或自动抓取与该股相关的新闻
- 关联到决策卡（"基于这条新闻做了什么操作"）

#### 3.3.3 移动止盈条件单（亮点功能）

针对盈利持仓，可以设置 **动态移动止盈线**：

```
案例：烽火通信，当前浮盈180%

设置：
├── 第一止盈：浮盈达到200%时，提醒减仓1/3
├── 第二止盈：浮盈从最高位回撤15%时，提醒减仓1/3  
└── 终极止损：浮盈从最高位回撤30%时，提醒全部清仓

App会推送提醒（不会自动交易，A股监管限制，
但会精确通知你"该执行预设动作了"）
```

### 3.4 模块③：📅 周度复盘

**功能定位**：每周日晚上15分钟的结构化反思。

#### 3.4.1 自动汇总（用户打开页面就能看到）

- 本周账户走势曲线（vs 沪深300/创业板/科创50）
- 本周所有交易卡片化展示
- 本周决策卡填写完整率
- 本周高危交易笔数（FOMO高/不符合体系/情绪化）
- 本周胜率（已有结果的交易）
- 本周操作次数 vs 自设上限

#### 3.4.2 周复盘三问（强制填写）

> 1. 本周做得最对的一件事？（必填，限100字）
> 2. 本周做得最错的一件事？（必填，限100字）
> 3. 如果重来一次，我会怎么做？（必填，限100字）

#### 3.4.3 纪律分（7项打分，满分14）

```
□ 没有满仓加杠杆                   0-2分
□ 没有追涨停/单日涨超7%的股票     0-2分
□ 单一持仓未超25%                  0-2分
□ 每笔操作都填了决策卡             0-2分
□ 没有盘中追高/恐慌杀跌            0-2分
□ 没有看股吧/荐股群                0-2分
□ 每周操作未超2次                  0-2分
```

**部分项可自动评分**（如"每笔操作都填了决策卡"由系统直接判断），部分需手动诚实打分。

#### 3.4.4 系统智能评估

- 纪律分 < 8 → 触发"暂停交易建议"
- 连续3周纪律分 < 10 → 触发"强制冷静期"提醒
- 跑赢大盘 → 自动恭喜并归因（"这周你的高纪律分对应了高收益"）

### 3.5 模块④：📆 月度画像

**功能定位**：每月一份个性化的"行为肖像"，让用户**真正看见自己**。

#### 3.5.1 月度数据卡

- 月度收益率 vs 大盘
- 最大回撤 + 回撤期间
- 交易次数 + 平均单笔金额
- 平均纪律分
- 决策依据分布饼图
- 情绪评分趋势线

#### 3.5.2 三笔关键交易

由系统辅助识别 + 用户确认：
- 🟢 **最成功的一笔**：盈利金额/盈利率最高
- 🔴 **最失败的一笔**：亏损金额/亏损率最高
- 🟡 **最反思的一笔**：盈亏不一定最大，但行为最值得反思（如FOMO高分却侥幸赚到）

对每笔交易追问：
> 这是新错误还是旧错误？（如选旧错误，关联到"错误类型库"）

#### 3.5.3 核心问题对照表（独创）

针对调研得出的散户**6大核心问题**，每月对照：

| 问题 | 本月评估 | 趋势 |
|---|---|---|
| 满仓+加杠杆 | ✅改善 / ➖持平 / ❌恶化 | (历史曲线) |
| 重仓追热点不研究 | ✅ / ➖ / ❌ | |
| 越跌越补、扩大亏损 | ✅ / ➖ / ❌ | |
| 赚的不卖、盈利变浮亏 | ✅ / ➖ / ❌ | |
| 凭情绪交易、被盘中波动主导 | ✅ / ➖ / ❌ | |
| 想短期回本、设定不切实际目标 | ✅ / ➖ / ❌ | |

12个月后形成一张完整的"6大问题进步曲线"。

#### 3.5.4 月度体悟 + 下月改进重点

- 本月体悟（自由文本，3-5句）
- **下月只选一个改进重点**（强制单选，不允许"全都要"）
- 衡量标准（如何判断这个重点是否完成）

### 3.6 模块⑤：📊 分析仪表盘（揭穿幻觉的真相之镜）

**功能定位**：用数据回答"我到底是什么类型的交易者？我真的擅长什么？我真的做错过什么？"

#### 3.6.1 七张核心图表

**图1：盈亏归因瀑布图**
- 按"决策依据类型"拆解你的盈亏
- 例：基本面交易贡献+5万、技术面+2万、情绪驱动-12万、看到别人买-3万
- **冲击力**：让你看到"凭感觉"的交易**净贡献是负的**

**图2：情绪评分 vs 实际盈亏 散点图**
- X轴：FOMO评分（1-10）
- Y轴：该笔交易的盈亏%
- **冲击力**：散点会清晰呈现"FOMO越高、亏损越大"的相关性

**图3：纪律分 vs 周收益 趋势图**
- 双轴折线图，左Y轴是纪律分，右Y轴是收益率
- **冲击力**：高纪律分的周通常对应正收益

**图4：错误重复频率热力图**
- X轴：错误类型（如"越跌越补"、"追高"、"不止盈"）
- Y轴：月份
- 颜色深浅：该月该错误发生次数
- **冲击力**：让用户清晰看到"哪个错误在结构性地重复"

**图5：单股操作复盘图**
- 选择一只股票，时间轴展示所有操作点（买/卖/加仓/减仓）
- 叠加K线图
- **冲击力**：直观看到"我在历史高点加仓、低点割肉"的实际表现

**图6：理性vs非理性决策对比**
- 把交易按决策依据分两组（理性5项 vs 非理性4项）
- 对比两组的：胜率、平均盈亏、累计贡献
- **冲击力**：99%的散户会发现非理性决策是亏损主因

**图7：账户净值曲线 vs 大盘**
- 类似雪球的对比图
- 关键节点标注（如重大失误日、重大盈利日）
- 计算 Alpha、Beta、最大回撤等专业指标

#### 3.6.2 智能洞察（每周/每月自动生成）

App用LLM对你的数据做自然语言总结：

> 示例洞察（针对调研用户）：
>
> "过去30天的关键发现：
> 1. 你在烽火通信上做得很好——共7次操作，6次盈利，平均盈利8%
> 2. 但在金风科技上重复了你的旧错误——越跌越补，单只仓位达40%
> 3. 你的'凭感觉'交易共3次，全部亏损，累计-1.2万
> 4. 建议下月重点：模仿烽火通信的操作模式（轻仓+波段+设条件单）"

### 3.7 模块⑥：🚨 智能预警

**功能定位**：用主动推送让你避免重复犯错。

#### 3.7.1 预警类型

| 类型 | 触发条件 | 推送内容示例 |
|---|---|---|
| 浮盈预警 | 持仓浮盈达到历史阈值 | "烽火浮盈达180%，建议执行预设第一止盈" |
| 浮亏预警 | 单股浮亏超10%/20% | "金风浮亏达10%，是否触发越跌越补的旧模式？" |

#### 3.7.2 个性化"反熵"提醒

基于该用户**自己的历史模式**，在关键时点推送：

> "周一开盘后2小时内是你历史上情绪化交易最高发的时间段（占全部冲动交易的47%）。今天的这2小时建议先做计划而不是操作。"

### 3.8 模块⑦：📚 错误类型库（个性化"病历本"）

**功能定位**：把每一次错误归类、记录、复盘，形成自己的"错误词典"。

#### 3.8.1 错误分类系统

预置常见错误类型：
- 追高（情绪驱动买入冲高股）
- 越跌越补（亏损股加仓）
- 不止盈（盈利股不卖等回撤）
- 频繁交易（短时间内反复操作）
- 凭感觉（无依据下单）
- 跟风（看到别人买卖跟随）
- 不止损（亏损股不卖等翻盘）
- ...用户可自定义

#### 3.8.2 错误记录方式

- 决策卡或月度画像中标注"这是一次错误" → 归类到错误库
- 系统自动累计同类错误的频次和总代价
- 每个错误形成卡片，记录：发生次数、累计金额损失、上次发生日期、典型场景

#### 3.8.3 错误回顾视图

按"成本最高的错误"排序：

| 错误类型 | 累计发生 | 累计代价 | 最近一次 | 趋势 |
|---|---|---|---|---|
| 越跌越补 | 11次 | -8.2万 | 2026/4/15 | 📉减少中 |
| 不止盈 | 8次 | -6.5万 | 2026/3/22 | 📈仍在重复 |
| 凭感觉买入 | 15次 | -3.1万 | 2026/5/8 | 📉减少中 |

### 3.9 模块⑧：🎯 目标管理

**功能定位**：把不切实际的目标（"1年回本"）转化为可执行的合理目标。

#### 3.9.1 目标设定向导

引导用户设定：
- 本金、当前账户、目标账户（如100万 → 100万）
- 时间周期（1年/2年/3年/5年）
- 系统自动算需要的**年化收益率**
- 与历史业绩对比：
  - 你过去6年年化：-3%
  - 沪深300长期年化：7-8%
  - 优秀基金经理：15-20%
  - 巴菲特长期：20%
- **现实性评分**：1-5⭐
- 不切实际的目标会**直接被劝退**（1年100%等）

#### 3.9.2 目标进度跟踪

- 月度进度条
- 偏离曲线（超额/不足）
- 智能调整建议

### 3.10 模块⑨：🧮 计算器套件

参考TraderWaves的Calculators模块，A股化定制：

- **仓位计算器**：根据账户、止损位、单笔最大亏损，算应该买多少股
- **凯利公式计算器**：根据胜率和盈亏比，算最优仓位
- **复利计算器**：本金+年化+时间→未来账户
- **税费计算器**：A股印花税、佣金、过户费
- **网格交易计算器**：高抛低吸网格规划
- **加仓决策器**：输入当前仓位+股价跌幅+基本面变化，输出"加仓建议"或"不加仓的理由"

## 五、UI/UX设计原则

### 5.1 设计语言

- **专业但不冰冷**：金融感+温度感
- **数据为王**：每屏一个核心指标，不堆砌
- **行动导向**：每个洞察后都有"下一步动作"按钮
- **移动优先**：60%使用场景在手机上

### 5.2 颜色规范

```
主色：深蓝 #1F4E78（专业、可信）
辅色：薄荷绿 #00C49A（正向）
警示：橙红 #FF6B35（负向、警告）
中性：灰阶 #2D3748 → #E2E8F0

数据色：
盈利：#00C49A
亏损：#FF4757
中性：#94A3B8

情绪色：

失控：深红 #FCA5A5
```

### 5.3 关键交互原则

1. **决策卡必须3屏内完成**（手机端单手操作）
2. **危险拦截必须显示用户自己的历史数据**（不是泛泛警告）
3. **每个图表都要有"这意味着什么"的文字解读**
4. **不允许"删除"行为数据**（错误不能被掩盖），只能"归档"
5. **重要操作的不可逆性**：核心问题对照表的历史评分不可修改
